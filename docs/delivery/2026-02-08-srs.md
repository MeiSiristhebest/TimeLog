# TimeLog Software Requirements Specification (Aligned Snapshot)

Date: 2026-02-08  
Baseline: Expo SDK 54 / RN 0.81 / TypeScript strict mode

## 1. Functional Requirements

| ID | Requirement | Priority | Implementation Anchor |
| --- | --- | --- | --- |
| SRS-SYNC-001 | System shall queue upload operations offline and replay when online. | P1 | `src/lib/sync-engine/queue.ts`, `src/lib/sync-engine/store.ts` |
| SRS-SYNC-002 | System shall queue metadata updates (title/delete state) and replay reliably. | P1 | `src/lib/sync-engine/queue.ts`, `src/features/story-gallery/services/storyService.ts` |
| SRS-SYNC-003 | System shall queue cloud file deletion (`delete_file`) for permanent deletion flow. | P1 | `src/lib/sync-engine/queue.ts`, `src/lib/sync-engine/store.ts`, `src/features/story-gallery/services/storyService.ts` |
| SRS-SYNC-004 | System shall record `delete_file` success/failure sync telemetry to Supabase `sync_events` with 90-day retention support. | P1 | `src/lib/sync-engine/metrics.ts`, `src/lib/sync-engine/store.ts`, `supabase/migrations/20260210_create_sync_events.sql` |
| SRS-REC-001 | Recording upload shall support Opus/WAV resolved upload asset selection. | P1 | `src/lib/sync-engine/transcode.ts`, `src/lib/sync-engine/store.ts` |
| SRS-STORY-001 | Story permanent delete shall remove local file and remove cloud object asynchronously. | P1 | `src/features/story-gallery/services/storyService.ts` |
| SRS-NTF-001 | Push registration shall request permission and persist Expo token in Supabase. | P1 | `src/lib/notifications.ts` |
| SRS-NTF-002 | Badge count shall reflect unread activities. | P2 | `src/lib/notifications/badgeService.ts` |
| SRS-NTF-003 | Nudge scheduling shall use inactivity threshold and local notifications. | P2 | `src/lib/notifications/nudgeService.ts` |

## 2. Non-Functional Requirements

| ID | Requirement | Priority | Notes |
| --- | --- | --- | --- |
| SRS-NFR-001 | Local-first writes must complete without network dependency. | P1 | Queue-based state transition pattern |
| SRS-NFR-002 | Sync operations must retry with exponential backoff. | P1 | `retryCount` + `nextRetryAt` |
| SRS-NFR-003 | TypeScript strict mode with no unchecked critical-path `any`. | P1 | Sprint 3 tightened sync payload typing |
| SRS-NFR-004 | Architecture boundary: `src/lib` shall not runtime-import `src/features`. | P1 | Enforced in Sprint 3 refactor |
| SRS-NFR-005 | Quality gate: lint/test/tsc shall pass before delivery. | P1 | Verified in Sprint 3 acceptance run |
| SRS-NFR-006 | CI shall fail on newly introduced ESLint warnings while preserving an explicit warning baseline. | P1 | `scripts/eslint-warning-budget.mjs`, `tools/eslint-warning-baseline.json`, `.github/workflows/ci.yml` |
