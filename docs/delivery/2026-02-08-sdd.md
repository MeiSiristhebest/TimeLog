# TimeLog Software Design Description (Sprint 3 Alignment)

Date: 2026-02-08

## 1. Design Summary

Sprint 3 focuses on delivery hardening:

1. Removed runtime architecture violations (`src/lib` importing `src/features`).
2. Added cloud deletion queue operation (`delete_file`) to close permanent-delete sync loop.
3. Added `delete_file` observability metrics (`sync_events`) with non-blocking failure handling.
4. Added CI warning budget enforcement using repository baseline.

## 2. Module Decisions

### 2.1 Sync Engine

- `sync_queue.type` now supports `delete_file`.
- Queue service adds `enqueueDeleteFile(recordingId, storagePath)`.
- Store processor handles `delete_file` by calling transport layer deletion.
- Transport adds `deleteFile(bucket, storagePath)` using Supabase Storage remove API.
- Store now records `delete_file` metrics:
1. success -> `delete_file_success`
2. failure -> `delete_file_failed`
- Metric write failures are swallowed and logged (`devLog.warn`) to avoid blocking retry/dequeue flow.

Files:
- `src/db/schema/syncQueue.ts`
- `src/lib/sync-engine/queue.ts`
- `src/lib/sync-engine/store.ts`
- `src/lib/sync-engine/transport.ts`
- `src/lib/sync-engine/metrics.ts`

### 2.2 Story Permanent Delete

- `permanentlyDeleteStory()` now:
1. deletes local file idempotently,
2. enqueues `delete_file` using derived cloud path `${recordingId}.${uploadFormat || 'wav'}`,
3. deletes local DB row.

File:
- `src/features/story-gallery/services/storyService.ts`

### 2.3 Boundary Cleanup

- Removed `src/lib/lazyComponents.tsx` (feature imports from lib).
- Introduced feature-local lazy binding:
  - `src/features/story-gallery/components/LazyTimelineLayout.tsx`
- Updated route usage in:
  - `app/(tabs)/gallery.tsx`
- Refactored notification services in `src/lib/notifications*` to avoid importing from `src/features/*`.

### 2.4 Warning Budget Gate

- Added script `scripts/eslint-warning-budget.mjs`.
- Baseline stored in `tools/eslint-warning-baseline.json`.
- CI runs baseline check after linter; new warning increases fail pipeline.

## 3. Data Contracts

Queue payloads:

- `upload_recording`: `{ filePath, uploadPath?, uploadExtension?, recordingId }`
- `update_metadata`: `{ recordingId, updates }`
- `upload_transcript_segment`: `TranscriptSegmentSyncPayload`
- `delete_file`: `{ recordingId, storagePath }`

Sync metrics row:
- `event_type`: `delete_file_success | delete_file_failed`
- `attempt`: queue retry attempt at event emission time
- `error_message`: sanitized and truncated summary only
- cleanup function: `cleanup_sync_events_older_than(days int default 90)`
